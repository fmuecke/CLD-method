# EVID-2026-002: BM↔ADO Epic bootstrap and linking verification

Story: ST-2026-002
Hypothesis: HYP-2026-001

## What Was Built

- `src/BmAdoSync.ps1` — added five functions:
  - `Get-SyncDelta` — pure function: computes paired / ADO-only / BM-only sets from two ID lists
  - `Get-BmBoardCards` — fetches all cards on a BM board via `GET /cards?board_ids=`
  - `Get-BmBoardColumns` / `Get-BmBoardLanes` — resolves column and lane IDs for card creation
  - `New-BmCard` — creates a BM card with title, column, lane, and custom_id
  - `Set-BmCardTitle` — updates BM card column/state via PATCH (used by state sync, not bootstrap)
  - `Get-AdoEpics` — WIQL query for non-Removed Epics in an ADO area path, batched in 200s
- `src/Invoke-BmAdoBootstrap.ps1` — orchestration script: delta → create missing BM cards → sync titles → report
- `tests/BmAdoSync.Tests.ps1` — 5 new unit tests for `Get-SyncDelta`

## What Was Tested

Unit tests for `Get-SyncDelta` (all 5 scenarios). Integration test skipped — requires live BM and ADO credentials.

## Expected vs Actual

| Scenario | Expected | Actual |

Test run: 16 passed (all unit tests), 1 not run (integration — requires credentials).

## Decision

- Status: **Partially confirmed** — delta logic verified. Integration path (BM API + ADO WIQL + card creation) not yet run against live systems.

## What Remains Uncertain

- BM API response shape for `GET /cards` — the `card_id`, `title`, `custom_id` field names assumed from spec; live call may differ.
- BM column/lane API endpoints (`/boards/{id}/columns`, `/boards/{id}/lanes`) — assumed path structure; needs verification.
- ADO WIQL field `System.StateCategory` in the WHERE clause — may not be filterable; may need to use `System.State` instead.
