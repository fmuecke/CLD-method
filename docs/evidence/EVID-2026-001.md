# EVID-2026-001: Compute BusinessMap Epic state from ADO child state categories

Story: ST-2026-001
Hypothesis: HYP-2026-002

## What Was Built

- `src/BmAdoSync.ps1` — three functions:
  - `Get-BmEpicState` — pure mapping function: list of ADO state categories → BM column name
  - `Get-AdoStateCategoryMap` — fetches state-name → category lookup for a work item type via ADO REST API
  - `Get-AdoChildStateCategories` — queries ADO for child items of a given Epic, resolves their state categories
- `tests/BmAdoSync.Tests.ps1` — Pester v5 test suite

## What Was Tested

Smallest possible test from ST-2026-001: unit tests for all five mapping scenarios, plus one integration test (skipped unless `ADO_PAT` env var is set).

## Expected vs Actual

| Scenario | Expected | Actual |
|---|---|---|
| All children Proposed | 'Requested' | 'Requested' ✓ |
| Any child InProgress | 'In Progress' | 'In Progress' ✓ |
| All active Complete (Removed excluded) | 'Done' | 'Done' ✓ |
| Mix of Complete and Proposed | 'In Progress' | 'In Progress' ✓ |
| All children Removed | $null (skip + log) | $null ✓ |

Test run: 11 passed, 0 failed, 1 skipped (integration — requires ADO_PAT).

## Decision

- Status: **Confirmed**
- The mapping function is correct for all five specified scenarios. Integration test is ready to run against a real ADO project once credentials are available.

## What Remains Uncertain

- `Get-AdoChildStateCategories` has not been run against a live ADO instance. The assumption that ADO's work item type state API returns a `stateCategory` field needs to be verified.
- Behavior when an Epic has no children at all is not yet specified (function returns empty array — caller behavior undefined).
